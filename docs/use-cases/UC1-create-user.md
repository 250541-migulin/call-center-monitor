# UC1: Создать учётную запись пользователя

## Актёр
Администратор

## Описание
Администратор создаёт нового пользователя системы с выбором роли (оператор, руководитель, администратор).

## Предусловия
- Администратор авторизован в системе.
- Открыт раздел «Управление пользователями».

## Основной поток событий
1. Администратор нажимает кнопку **«Добавить пользователя»** на странице управления пользователями.
2. Система открывает модальное окно или новую форму с полями:
   - **Логин** (текстовое поле, обязательное),
   - **Роль** (выпадающий список: «Оператор», «Руководитель», «Администратор»).
3. Администратор вводит уникальный логин и выбирает роль.
4. Администратор нажимает кнопку **«Сохранить»**.
5. Система проверяет корректность данных:
   - логин не пустой,
   - логин не дублируется.
6. Система создаёт учётную запись с временным паролем (например, `Temp123!`) и отправляет уведомление (опционально).
7. Система отображает сообщение: **«Пользователь успешно создан»** и обновляет список пользователей.

## Постусловия
- Новый пользователь может войти в систему с указанным логином и временным паролём.
- При первом входе система требует смены пароля (если реализовано).

## Альтернативные потоки

### A1: Логин уже существует
1. На шаге 5 система обнаруживает, что логин уже используется.
2. Система отображает ошибку под полем логина: **«Пользователь с таким логином уже существует»**.
3. Форма остаётся открытой; администратор может изменить логин и повторить попытку.

### A2: Логин не указан
1. Администратор оставляет поле «Логин» пустым.
2. При нажатии «Сохранить» система подсвечивает поле красным и показывает: **«Логин обязателен»**.
3. Кнопка «Сохранить» не отправляет данные до заполнения всех обязательных полей.

### A3: Роль не выбрана
1. Администратор не выбирает роль (оставляет значение по умолчанию или пустое).
2. Система блокирует отправку и выводит: **«Выберите роль пользователя»** рядом с выпадающим списком.

### A4: Отмена операции
1. Администратор нажимает кнопку **«Отмена»** или закрывает модальное окно.
2. Система закрывает форму без сохранения изменений.
