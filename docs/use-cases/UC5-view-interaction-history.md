# UC5: Просмотреть историю обращений клиента

## Актёр
Оператор, Руководитель

## Описание
Пользователь просматривает полную историю взаимодействий с конкретным клиентом, включая даты, темы, ресурсы, результаты и комментарии.

## Предусловия
- Клиент существует в системе (имеет карточку).
- У клиента есть хотя бы одно зафиксированное обращение.

## Основной поток событий (Flow of Events)
1. Пользователь переходит в раздел «Клиенты» или открывает карточку клиента из журнала звонков.
2. Система отображает основные данные клиента: имя, номер телефона, дату первого и последнего звонка.
3. Пользователь нажимает кнопку **«Показать историю обращений»**.
4. Система загружает все обращения клиента из базы данных.
5. Система отображает список обращений в **хронологическом порядке** (от старых к новым или наоборот, с возможностью сортировки).
6. Каждое обращение содержит:
   - Дату и время,
   - Ресурс,
   - Тему обращения,
   - Результат («Решено» / «Не решено»),
   - Комментарий оператора,
   - Имя оператора, обработавшего обращение.

## Альтернативные потоки
- **A1**: Если у клиента нет обращений, система показывает сообщение:  
  > «Нет записей в истории обращений».

## Постусловия
- Пользователь получает полную картину взаимодействия с клиентом.
- Информация может быть использована для принятия решений (например, выявить частого клиента с проблемами).

## Примечание
Этот Use Case доступен как **оператору** (для работы с текущим клиентом), так и **руководителю** (для анализа качества обслуживания).
