# UC7: Отфильтровать данные по параметрам

## Актёр
Руководитель

## Описание
Руководитель применяет фильтры к журналу звонков и обращений, чтобы сузить выборку данных по заданным критериям.

## Предусловия
- В системе имеются зафиксированные звонки и обращения.
- Руководитель находится на странице с таблицей данных (например, «Все звонки» или «История обращений»).

## Основной поток событий (Flow of Events)
1. Руководитель открывает панель фильтрации (обычно расположена над таблицей).
2. Система отображает доступные фильтры:
   - **Дата**: диапазон (от – до),
   - **Оператор**: выпадающий список зарегистрированных операторов,
   - **Клиент**: поле ввода номера телефона или имени,
   - **Ресурс**: чекбоксы (`portal.example.com`, `api.example.com`, `mobile-app`),
   - **Статус обращения**: радиокнопки (`Решено`, `Не решено`).
3. Руководитель выбирает один или несколько параметров.
4. Руководитель нажимает кнопку **«Применить фильтр»**.
5. Система обновляет таблицу, отображая только записи, соответствующие выбранным критериям.
6. Система сохраняет текущие параметры фильтрации в URL (для возможности поделиться ссылкой).

## Альтернативные потоки
- **A1**: Если по заданным критериям нет данных, система показывает сообщение:  
  > «Нет записей, соответствующих выбранным фильтрам».

## Постусловия
- Руководитель получает узкую выборку данных, соответствующую его запросу.
- Выборка может быть использована для анализа, экспорта (UC8) или детального просмотра.

## Примечание
Фильтрация применяется **ко всем типам данных**: звонкам, обращениям, статистике. Это ключевой инструмент для гибкой аналитики.
